
# Dimensional Escalation Playbook (v1) — 2025-10-04T01:34:29Z

> 原則：遇到難題，不找藉口；換維度、再回來。所有步驟**可逆**，每次行動都留下 trace。

## 六層解題座標
1. **資料層 (D)**：檔案正確性、編碼、欄位、單位、指標可計算。
2. **結構層 (S)**：樹/圖/索引/跳點一致；.fltnz/.flpkg/.map 是否對上。
3. **流程層 (P)**：吸收→標準化→對齊→融合→進化 是否跑通；紀律是否落地。
4. **原理層 (R)**：守恆/對稱/無量綱化；五因子+溫度 是否支撐當前決策。
5. **綜合法 (C)**：跨模組/跨平台對齊（iOS/Windows/雲端），快照沙盒能否單獨運轉。
6. **元層 (M)**：目標函數/限制重構；定義何者為「完成」、如何量測；必要時縮減問題。

## 升維觸發條件
- D 層：讀不到/亂碼/副檔名不符/空欄>5%。
- S 層：索引破損/路徑缺失/跳點找不到。
- P 層：任一階段輸出與預期 schema 不符。
- R 層：物理/邏輯不一致（違背守恆/對稱）、因子失衡。
- C 層：跨端結果不一致、快照不可播。
- M 層：目標函數與現況不相容、成功定義不清。

## 降維回補
- 從高層取得「約束/重定義」後，回寫到低層：
  - M→R：重定義成功函數→更新守恆/對稱列表。
  - R→P：把新的約束寫進流程驗證點。
  - P→S：用流程產出的 map 修復索引。
  - S→D：批次修復副檔名/編碼/欄位。

## 公式/規則（可帶入 Discipline 檔）
- **注意力分數**： s = 0.35·Structure + 0.25·Topic + 0.20·Persona + 0.10·Recency + 0.10·Cred
- **溫度**： T=0.85（收斂）；T=1.30（探索）
- **放大/折損**： P_{k+1} = N_k · P_k · η_k
- **可逆紀錄**： 每次輸出都寫：trace/、.flynz.map、discipline.json

## 標準操作（SOP）
1) 先 D→S：`Auto-Audit` 修復檔名/編碼→重建索引與跳點。
2) 再 P：吸收/對齊/融合/進化 全跑一次；失敗則升 R。
3) R：套守恆/對稱/無量綱化；不過則升 C。
4) C：跨端掛載（iOS/Windows/雲端）驗證快照自足；仍不過則升 M。
5) M：重定義成功與邊界，縮減問題；把新約束回寫 R/P/S/D。
